Role-Based Directives Manual Test Guide

RoleBasedShowDirective

This directive conditionally shows/hides elements based on user role and optional market access.

Usage Examples:

<!-- Single role -->
<div *roleBasedShow="UserRole.Admin">Admin Only Content</div>

<!-- Multiple roles -->
<div *roleBasedShow="[UserRole.CM, UserRole.Admin]">CM or Admin Content</div>

<!-- With market filtering -->
<div *roleBasedShow="UserRole.CM; market: 'NYC'">NYC CM Content</div>

Features:

  • ✓ Structural directive for conditional rendering
  • ✓ Supports single or multiple roles
  • ✓ Optional market parameter for market-specific visibility
  • ✓ Automatically updates when login status changes
  • ✓ Checks both role and market access

RoleBasedDisableDirective

This directive conditionally disables elements based on user role.

Usage Examples:

<!-- Single role -->
<button [roleBasedDisable]="UserRole.Admin">Admin Only Action</button>

<!-- Multiple roles -->
<button [roleBasedDisable]="[UserRole.CM, UserRole.Admin]">CM/Admin Action</button>

<!-- Custom message -->
<button 
  [roleBasedDisable]="UserRole.Admin"
  [roleBasedDisableMessage]="'Only admins can delete users'">
  Delete User
</button>

Features:

  • ✓ Attribute directive for conditional disabling
  • ✓ Supports single or multiple roles
  • ✓ Adds visual styling (opacity, cursor, pointer-events)
  • ✓ Adds tooltip explaining why element is disabled
  • ✓ Custom tooltip message support
  • ✓ Adds aria-disabled for accessibility
  • ✓ Works with form controls and non-form elements
  • ✓ Automatically updates when login status changes
